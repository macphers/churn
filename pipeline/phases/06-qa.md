## QA Process

### 1. Smoke Test

Can the app start without errors?
- Start the application
- Check for console errors
- Verify the main page/screen renders

If the app won't start, that's a critical bug. Log it and stop.

### 2. Success Criteria Verification

Read the success criteria from `{PHASE_ARTIFACTS}/phase-01-plan-ceo.md`.
Test each criterion literally:

For each criterion:
- Attempt the exact user action described
- Record PASS or FAIL
- If FAIL: describe what happened vs. what should have happened
- Take a screenshot of the result

### 3. Edge Case Testing

For each user-facing feature:
- **Empty input:** What happens?
- **Invalid input:** What happens?
- **Rapid repeated actions:** What happens?
- **Browser back button:** What happens? (if web app)
- **Refresh during operation:** What happens? (if web app)

### 4. Visual Inspection (Web Apps)

If the app has a UI, use the browse binary:

```bash
$B goto "http://localhost:3000"  # or whatever port
$B snapshot -a                    # annotated screenshot
$B screenshot "{PHASE_ARTIFACTS}/screenshots/main-page.png"
```

Check:
- Does it look broken? Overlapping elements, missing styles?
- Is text readable? Sufficient contrast?
- Does it work on a narrow viewport?
  ```bash
  $B viewport 375 812
  $B screenshot "{PHASE_ARTIFACTS}/screenshots/mobile.png"
  $B viewport 1280 720
  ```

### 5. Test Suite Verification

Run the test suite and record results:
```bash
cd output && npm test 2>&1
```

All tests should pass. A failing test is a bug.

## Bug Classification

For each bug found:
- **Critical:** App crashes, data loss, security hole, success criteria fails
- **Major:** Feature doesn't work as expected but app doesn't crash
- **Minor:** Visual glitch, typo, rough edge
